Project Roadmap

(i)Phase 1: Dataset Understanding (EDA)
Objective: Understand the dataset before touching any model.
We'll answer questions like:

How many rows and columns?
What is the target column?
Are there missing values?
How many ticket categories?
Is the dataset balanced?
What are the most frequent words?
Ticket length distribution.
Correlation between features (where applicable).

Deliverable:
Clean dataset
EDA notebook
Business understanding

(ii)Phase 2: Data Preprocessing
Apply everything you've learned:

Lowercasing
Remove punctuation
Remove numbers (if required)
Remove special characters
Remove stopwords
Lemmatization

We'll also discuss why we're doing each step.

Deliverable:
Clean text ready for ML

(iii)Phase 3: Feature Engineering
Convert text into numerical vectors.

We'll compare:
CountVectorizer (for comparison)
TF-IDF (main model)

We'll also decide:

max_features
ngram_range
min_df
max_df

These are important hyperparameters that interviewers often ask about.

(iv)Phase 4: Model Building

We'll train multiple models.

Logistic Regression
Multinomial Naive Bayes
Random Forest

Instead of assuming one is best, we'll compare them.

(v)Phase 5: Model Evaluation

We'll compare:

Accuracy
Precision
Recall
F1 Score
Confusion Matrix
Classification Report

Then choose the best model.

(vi)Phase 6: Priority Prediction

We'll repeat the same pipeline for the priority column.

This gives us two trained models.

(vii)Phase 7: Resolution Recommendation

We'll use:

TF-IDF
Cosine Similarity

to find similar tickets and display the corresponding resolution_notes.

No RAG yet—this is classic information retrieval.

(viii)Phase 8: Business Logic

We'll map categories to support teams.

Example:

{
    "Payment Problem": "Finance Team",
    "Login Issue": "Authentication Team",
    ...
}

(ix)Phase 9: Streamlit Deployment
The final app will look like:

Enter Customer Issue
↓
Category
↓
Priority
↓
Assigned Team
↓
Suggested Resolution
